# Clay
A documentation framework for developers, built on [Nuxt.js](https://nuxt.com/) and [Vue.js](https://vuejs.org/). Clay provides a structured, auto-generated system for displaying documentation written in **Markdown**.

> Clay is **not** a static site generator. It is a documentation framework that integrates into existing projects with minimal effort.

## Capabilities
- Auto-generates documentation structure from the contents of the `docs/` directory.
- Modern UI via **[daisyUI](https://daisyui.com/)** and **[Tailwind CSS](https://tailwindcss.com/)**.
- Rich Markdown parsing via **[marked.js](https://marked.js.org/)** and syntax highlighting via **[Shiki](https://shiki.matsu.io/)**.
- Customizable through a single `clay.yaml` config file.

## How It Works
The Clay frontend is pre-built and loads everything it needs from a couple of config files, so it drops into any repository with minimal effort.

```text
.
├── docs/                    # Documentation directory (Markdown)
│   ├── my-doc.md
│   └── my-sub-directory/
│       └── nested-doc.md
├── clay.yaml                # Main configuration file
├── clay-structure.yaml      # Structure file (generated by Clay Oven)
└── logo.svg                 # Optional logo
```

## Configuration
`clay.yaml` configures the title, navbar, languages, and landing page:

```yaml
title: "Demo"
favicon: "logo.svg"
baseURL: "/"
navbar:
  logo: "logo.svg"
  links:
    - name: "Homepage"
      icon: "fa-solid fa-house"
      link: "https://example.com"
index:
  title: "Welcome to the Demo"
  icon: "fa-solid fa-rocket"
langs:
  - "python"
  - "js"
```

`clay-structure.yaml` defines the navigation tree and is generated automatically by **Clay Oven**, the companion Go build tool.

## The Clay Toolchain
- **Clay** — the Vue/Nuxt documentation frontend.
- **Clay Oven** — a Go CLI that scans your docs, generates structure files, and bundles them with a Clay distribution into a ready-to-deploy site.
- **Clay Example Repo** — a fully working example project.
